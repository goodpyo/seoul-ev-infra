# 서울시 전기차 충전 인프라 분석 — 발표 핵심 정리

> 데이터 기준: 전기차 등록 현황(2026.04) · 충전소 설치현황(2026.05.12)
> 작성 메모용 — 발표 자료는 추후 제작

---

## 1. 한 줄 핵심 메시지

> **"서울의 충전 인프라는 평균적으로 충분하다. 그러나 행정동 단위로 보면 충전 사각지대가 숨어 있다."**

자치구 단위 평균은 모든 구가 '충분'이라 문제가 안 보이지만,
행정동 단위로 쪼개면 **15개 동(3.5%)** 에서 부족이 드러난다 — 이것이 본 분석의 핵심 발견.

> ⚙️ **데이터 정책 (최종)**: 충전기 데이터는 **한국환경공단 API 단일 소스** 사용.
> 엑셀 보완분(586기, 0.8%)은 GPS가 없어 행정동 매핑 노이즈의 주원인이라 제외함 (§8 참고).

## 2. 서울 전체 현황 (총량)

| 항목 | 수치 |
|---|---|
| 전기차 등록 | **111,761대** |
| 충전기 총계 | **69,770기** (급속 5,532 / 완속 64,238) |
| 평균 부족지수 | **1.6** (충전기 1기당 전기차 대수) |

→ 총량으로는 충전기 1기당 전기차 1.6대. 절대량은 충분한 편.

## 3. "평균의 함정" — 단위에 따라 결론이 뒤집힌다

### 자치구 단위
- 25개 구 부족지수 **1.1 ~ 2.8** → **전부 '충분'**
- 가장 높은 구조차 강남구 2.8 (≤5 = 충분)
- 자치구 지도만 보면 "서울에 문제 없음"으로 오해

### 행정동 단위 (전기차 등록 427개 동 기준)

| 상태 | 동 수 | 비율 |
|---|---|---|
| 충분 | 373개 | **87.4%** |
| 보통 | 38개 | 8.9% |
| 부족 | 13개 | 3.0% |
| 매우 부족 | 2개 | 0.5% |
| 충전소 없음 | 1개* | 0.2% |

→ **부족 이상 = 15개 동 (3.5%)**. 평균에 가려진 사각지대.
> *'충전소 없음' 1개는 상일동 — 행정구역 개편 잔여 버킷이라 분석 제외 (§5 참고)

## 4. 강남구 사례 — 평균의 함정을 보여주는 대표 예시

- **자치구 평균**: 전기차 15,631대 / 충전기 5,644기 → 부족지수 **2.8 (충분)**
- 그러나 동별 편차가 큼:
  - 개포1동: 부족지수 **16.8 (부족)** ← 같은 강남이지만 부족
  - 도곡2동: 부족지수 **0.8 (충분)**
- 강남구 전체로는 충분해 보여도, 동 단위로는 부족 지역이 존재

> 발표 포인트: "강남도 동별로 보면 부족한 곳이 있다" → 자치구 평균의 한계를 직관적으로 전달

## 6. 지표 정의 (발표 시 설명용)

- **부족지수 = 전기차 등록 수 ÷ 충전기 수** (충전기 1기가 감당하는 전기차 대수)
  - 값이 클수록 부족
  - 매우 부족(>20) / 부족(>10) / 보통(>5) / 충분(≤5)
- **충전편의지수 = (급속×3 + 완속) ÷ 전기차 × 100** (급속에 가중치)

### 지표 설계 근거 (Q&A 대비 — "왜 이렇게 정했나?")

**① 부족지수를 '전기차÷충전기'로 정한 이유**
- 국제적으로 충전 인프라 수준을 비교할 때 쓰는 표준 지표가 **EV/충전기 비율**
  - IEA(국제에너지기구) Global EV Outlook: 국가별 충전 인프라를 "공용 충전기 1기당 전기차 대수"로 비교
  - EU AFIR(대체연료인프라규정) 권고: **전기차 10대당 공용 충전기 1기** (= 부족지수 10)
- 직관적 해석 가능: "충전기 1기를 전기차 몇 대가 나눠 쓰는가"

**② 임계값 5 / 10 / 20의 근거**
- **≤5 (충분)**: 한국 전국 평균 수준 — 한국은 충전기 1기당 전기차 약 1.7~2.0대(2024.10 기준 전기차 65만 대/충전기 39.4만 기)로 **IEA 조사 대상 30여 개국 중 1위**. 서울 평균도 1.6. 5 이하면 평균 대비 양호
- **>10 (부족)**: EU 권고 기준(10대당 1기)을 초과 → 국제 권고선 미달
- **>20 (매우 부족)**: 국제 권고 기준의 2배 초과 → 명백한 사각지대
- 즉 절대 기준(국제 권고) + 상대 기준(서울 평균 1.6의 수 배~십수 배)을 함께 반영

**③ 편의지수에서 급속×3 가중치의 근거**
- **법적 구분 기준**: 친환경자동차법 시행령상 급속 = 최대 출력 **40kW 이상**, 완속 = 40kW 미만
- **처리 능력 차이**: 급속(40~100kW+)은 1회 충전 약 30~40분, 완속(7kW)은 5~7시간 → 하루에 처리 가능한 차량 대수가 급속이 수 배 이상
- 출력 기준으로만 보면 40kW÷7kW ≈ **5.7배**지만, 급속은 80%까지만 충전하는 관행·대기시간 등을 감안해 **보수적으로 3배** 채택
- **×100**: 값이 너무 작아지지 않도록 한 스케일 조정일 뿐 (해석에 영향 없음)

**③-1 참고: 우리나라 공식 기준 (질문 대비)**
- 우리나라 법정 기준은 'EV 대수당 충전기'가 아니라 **주차면수 기준**: 신축시설 총주차대수의 **5%** 이상, 기축시설 **2%** 이상 충전시설 의무설치 (친환경자동차법 시행령, 세부 비율은 시·도 조례 위임)
- 서울시 조례: 의무설치 충전시설 10기 이상 시 **1기 이상은 급속** 설치
- → 본 분석은 '설치 의무' 관점이 아니라 '수요(등록 EV) 대비 공급' 관점이므로 EV/충전기 비율 지표를 채택, 임계값은 국제 권고와 서울 평균을 참조한 조작적 정의

**④ 한계 인정 (선제 답변용)**
- 임계값 자체는 공식 표준이 아니라 위 근거를 바탕으로 한 본 연구의 조작적 정의
- 충전기 이용률(가동 시간)·주행거리 차이는 반영 안 됨 → 향후 실시간 이용 데이터 결합 시 개선 가능

## 7. 정책 시사점 (결론 방향)

1. **자치구 단위 정책의 한계** — 구 평균으로는 사각지대를 못 잡음. 행정동 단위 접근 필요.
2. **우선 투자 15개 동** — 특히 둔촌1동·중곡 일대·대치1동(고수요)에 충전기 우선 확충.
3. **수요 규모 함께 고려** — 부족지수가 같아도 대치1동(3,428대)처럼 절대 수요가 큰 곳을 우선.

### 프로젝트 동기로 돌아가는 결론 ("전기차 사도 될까?")

본 프로젝트의 출발점은 **"전기차를 사보려는데 충전 인프라가 괜찮은가?"**라는 실제 구매 고민이었음.

- **거시적 답**: 한국은 충전기 1기당 전기차 약 2대로 세계 1위, 서울 평균 부족지수 1.6 → 평균적으로는 충전 걱정 없이 구매 가능
- **단, 평균의 함정**: 행정동 단위로는 부족지수 20 초과 사각지대 존재 (중곡1동: 전기차 101대에 완속 5기·급속 0기)
- **실용적 결론**: 구매 전 우리 앱으로 **자기 동네 부족지수 + 가까운 충전소 3곳**을 확인하면 됨 → 분석이 실제 구매 의사결정 도구로 연결
- 단서 2가지: ① 공공 충전기 기준이므로 거주 단지 내 전용 충전기가 있으면 실제 여건은 더 좋음 ② 분석 범위가 서울 한정 → 수도권 확장이 자연스러운 향후 과제

## 8. 데이터 신뢰도 검증 결과 (중요 — 질문 대비)

행정동 단위 부족 데이터가 파싱 오류로 왜곡됐는지 **GPS 실측으로 직접 검증함.**

### 검증 방법
- 충전기 GPS 좌표를 행정동 경계 폴리곤에 점-포함(point-in-polygon) 검사 → 실측 충전기 수 산출
- 파이프라인 집계값과 대조

### 결과
- **TOP 부족 동 15곳 중 GPS 검증 가능한 14곳 모두 충전기 수 정확히 일치(±1)** → 부족 순위는 신뢰 가능
- 단, 두 가지 정밀도 한계 발견:

**(1) 행정동 미배정 충전기 → 엑셀 보완분 제외로 대폭 축소 (1,043기 → 216기)**
- 당초: 미배정 1,043기(1.5%) 중 대부분(987기)이 **GPS 없는 엑셀 보완 데이터**가 `둔촌동·반포동` 등 **법정동명**으로 라벨된 케이스
- **최종 결정: 엑셀 보완분(586기, 0.8%) 데이터에서 제외** → API 단일 소스(GPS 기반 행정동 정밀 매핑)로 통일
  - 근거: 정확도 검증된 API 대비 엑셀은 GPS가 없어 매핑 노이즈의 주원인. 0.8% 보완 효과보다 신뢰도 훼손이 더 큼
- 결과: 미배정이 **216기(0.3%)** — API 데이터 중 GPS 누락/오류 좌표분만 남음
- 영향 방향: 일부 동의 부족이 미세하게 **과장**될 수 있음 (부족이 실제보다 약하게 나올 일은 없음 — 보수적 오차)
- 등급 변동 가능 동: 용산2가동·반포본동·독산4동·신월6동 (⚠️ 표시 유지)
- **둔촌1동·대치1동·중곡 일대·개포1동 등 11곳은 보정해도 부족 유지 → 확실**

**(2) 상일동 미세 버킷**
- `상일동`(41대), `상일제1동`(607대), `상일제2동`(157대)이 별도 버킷으로 공존 (파싱 오류 아님)
- `상일동` 41대 버킷만 부족지수 20.5로 튀어 보이나, 상일 지역 전체는 충분 → TOP에서 제외

### 지표 자체의 한계
- 부족지수는 **완속 충전기(아파트 등)까지 1기로 동일 카운트** → 실제 체감 부족과 차이 가능
- 충전기 69,770기 중 **완속이 64,238기(92.1%)** → 급속만 따로 보면 부족 지역이 더 늘어날 가능성

### 후속 개선 제안
- 잔여 미배정 216기는 주소 지오코딩(카카오 API)으로 좌표 보강 가능
- 제외한 엑셀 보완분도 지오코딩으로 GPS를 부여하면 재편입 가능 (0.8% 회복)
- **급속 충전기 한정 부족지수** 별도 산출 → 더 현실적인 사각지대 도출

### 분석 대상 범위 (질문 대비)

**본 분석은 "공용(외부 개방형) 충전기"만 대상으로 한다.**

| 포함 O | 포함 X |
|--------|--------|
| 공영주차장 충전기 | 아파트 입주민 전용 충전기 |
| 마트·백화점 공용 충전기 | 회사 직원 전용 충전기 |
| 도로변 급속 충전소 | 개인 가정용 충전기 |
| 공공기관 개방 충전기 | |

- 한국환경공단 API와 전기차 충전소 설치현황 엑셀 모두 **공용 등록 충전기**만 수록
- 아파트 지하주차장 충전기는 대부분 입주민 전용이라 공단 DB에 미등록 → 분석에서 누락
- ⚠️ 단, **아파트라고 일괄 제외되는 게 아님** — 등록 방식에 따라 갈림 (Q&A 대비)
  - 포함 예: 상도 롯데캐슬파크웰 B4 (등록 충전사업자가 공용으로 등록 → 데이터에 존재)
  - 미포함 예: 올림픽파크포레온 (입주민 전용/미등록 → 데이터에 없음)
  - 기준은 "아파트 여부"가 아니라 **환경공단 정보망에 공용 충전기로 등록됐는지 여부**

> **둔촌1동(1위) 해석 주의**: 올림픽파크포레온(12,032세대)은 단지 내 전용 충전기가 별도 존재하므로,
> "공공 충전 인프라 부족"이지 "충전기 자체가 없는 것"과는 다름.
> 대규모 신축 단지가 밀집한 동은 실제보다 인프라가 부족하게 나타날 수 있음.

**발표 권장 표현**:

## 9. "내 주변 충전소 — 실시간 현황" 미작동 원인 (질문 대비)

`getChargerStatus`(공공데이터포털 개방 API)로 실시간 충전 상태를 받아오려 했으나 표출 실패.
직접 API를 여러 차례 테스트해 원인을 규명함.

### ❗ 결론: "일 1,000건 한도 초과"가 원인이 **아님**
- 응답 코드가 `resultCode: "00"` (**정상**) — 할당량 초과 코드(`22`)나 인증 오류(`30`)가 아니었음
- 할당량 초과면 즉시 에러 응답이 오지만, 실제 증상은 **응답 지연/타임아웃**으로 양상이 다름
- (참고) 개발계정 일 1,000건 한도는 존재 → 다수 사용자 대상 서비스라면 별도 제약이지만, 현재 미표출의 직접 원인은 아님

### 실제 원인: 개방 API의 성능·안정성 한계

**(1) 응답이 극도로 느림 (호출당 약 21초)**
- 단일 충전소 조회에 15~25초 소요. 앱 타임아웃(초기 5초)에 전부 걸림 → 25초로 상향했으나 한계
- 실시간 웹 표출(여러 충전소 동시 조회)에는 사실상 부적합한 속도

**(2) 동시 요청을 못 견딤**
- 충전소 150곳을 동시(8스레드) 조회 → **100% 타임아웃**
- 부하가 쌓이면 단일 순차 조회조차 응답 불능 상태가 됨 (테스트 중 확인)

**(3) 등록된 실시간 데이터도 제한적**
- API가 응답할 때도 상당수 충전소가 `totalCount: 0`(실시간 기록 없음) 반환
- 운영사별 편차 큼. 실시간 제공 확인 운영사 예: **ST**(에스트래픽), **ME**, **CR**, **KO** 등
- ※ 정확한 커버리지 비율은 API가 너무 불안정해 신뢰성 있게 측정하지 못함

### ev.or.kr에서는 왜 실시간이 잘 나오나?
- ev.or.kr은 **한국환경공단 직영 사이트** → 충전사업자와의 **내부 실시간 연계 데이터**를 사용
- 우리가 쓴 **공개 개방 API(`getChargerStatus`)는 그 내부 데이터와 별개**로, 속도·안정성·커버리지가 떨어짐
- 즉 "데이터가 세상에 없는 것"이 아니라, **공개 API 경로의 품질이 내부 시스템에 못 미치는 것**

### 최종 결정: 실시간 기능 제거
- 공개 API의 구조적 한계(응답 21초, 동시 요청 불가)로 서비스 불가능 판단
- 실시간 조회 코드 전면 제거 → 급속/완속 **정적 설치현황만 표시**로 단순화
- 덕분에 주소 검색 후 결과 표출 속도가 즉시(1~2초)로 개선됨

### 한 줄 요약 (발표용)
> "실시간 충전 현황은 **공개 개방 API의 응답 속도(~21초)·안정성 한계**로 표출이 어려움.
> ev.or.kr은 환경공단 **내부 실시간 연계**를 쓰기 때문에 가능한 것이며,
> 우리가 접근 가능한 공개 API는 그 품질에 못 미침. (개발계정 호출 한도 문제가 아님)"

## 10. "가까운 충전소 찾기" 주소 검색 개선 (기술 구현 관련)

### 기존 방식: Nominatim (OpenStreetMap 무료 서버)
- 해외 서버 경유로 응답 1~3초 이상
- 건물명("금천구청" 등) 검색 정확도 낮음
- "서울시"를 "서울특별시"로 인식 못 하는 경우 있음

### 개선: 카카오 로컬 API로 교체
- 카카오 개발자 콘솔에서 앱 생성 → REST API 키 발급 (무료, 일 30만 건)
- 응답속도 ~0.3초, 한국 주소·건물명 정확도 높음
- 주소 검색 실패 시 키워드 검색(장소명)으로 자동 폴백
- "서울시" → "서울특별시" 자동 정규화 처리
- API 키는 `.env` 파일로 분리하여 GitHub에 노출 방지
- 카카오맵 API 활성화 + 플랫폼(Web) 등록 필요 (`localhost:8501`)
- Nominatim은 카카오 실패 시 2차 폴백으로 유지

### 반경 내 충전소 탐색도 최적화
- 기존: `apply()` 파이썬 루프로 7만 건 순차 계산 → 느림
- 개선: numpy 벡터 연산으로 전체 일괄 계산 → 즉시 처리

### UX 개편: "반경 100m" → "가장 가까운 3곳" (최종 발표자료 제작 후 변경)
- **검색 방식 변경**: 반경 100m 내 전체 표시 → 거리순 **TOP 3 충전소만** 표시
  - 반경 방식은 충전소가 0곳이거나 수십 곳이 나와 정보량이 불안정 → 항상 3곳 보장
- **번호 마커**: 지도에 1·2·3 번호 마커(파랑/초록/보라)로 거리 순위 직관 표시
- **카드 ↔ 지도 연동**: 우측 충전소 카드의 📍 버튼 클릭 시 지도가 해당 충전소 중심으로 이동·확대
- **자동 이동**: 검색(주소/GPS 자동감지) 직후 가장 가까운 1번 충전소로 지도 자동 이동 + 확대(zoom 17), "전체 보기" 버튼으로 3곳 조망 복귀
- **30m 클러스터링**: 급속/완속이 별도 등록된 동일 충전소(30m 이내)는 카드 1장으로 병합
- **버그 수정**: GPS 자동 감지 시 rerun 타이밍에 따라 검색 결과가 누락되던 문제 → 좌표 확보 시 매 실행마다 즉시 계산(벡터 연산이라 수 ms)으로 변경
- **서울 외 지역 한계**: 주소 검색은 서울로 강제 정규화됨 — 서울 외 주소(예: 인천 부개동)는 검색 불가 (데이터 자체가 서울 한정이므로 의도된 동작)

## 13. 프로젝트 수행 중 어려웠던 점 & 해결 과정

### 🔴 난관 1. 행정동 vs 법정동 혼재 문제
**문제**: 충전소 주소가 "둔촌동", "반포동" 등 법정동명으로 표기되어 있어, 행정동(둔촌1동 등)으로 자동 배정이 안 됨. 전체 충전기의 1.5%(1,043기)가 미배정 — 대부분 GPS 없는 엑셀 보완 데이터.

**해결**: ① 충전기 GPS 좌표를 행정안전부 행정동 경계 GeoJSON 폴리곤에 point-in-polygon 검사로 실측 매핑. ② GPS가 없어 매핑 불가한 엑셀 보완분(586기, 0.8%)은 신뢰도 우선 원칙으로 **최종 제외** → 미배정이 216기(0.3%)로 축소되고 데이터 소스가 API로 단일화됨.

---

### 🔴 난관 2. 실시간 충전 현황 API 미작동
**문제**: 공공데이터포털 `getChargerStatus` API로 실시간 충전 상태를 표출하려 했으나, 단일 충전소 조회에 15~25초 소요. 동시 요청 시 100% 타임아웃. 서비스 불가 판단.

**해결**: 원인을 "일 1,000건 한도"로 오해할 수 있으나 `resultCode: "00"` 정상 응답 확인으로 아님을 규명. ev.or.kr은 내부 연계 데이터 사용이라 가능한 것. 실시간 기능 전면 제거하고 정적 설치현황으로 대체 → 오히려 앱 응답속도 개선.

---

### 🔴 난관 3. 주소 검색 속도 문제
**문제**: Nominatim(OpenStreetMap 무료 서버) 사용 시 응답 1~3초 이상, 건물명("금천구청" 등) 검색 정확도 낮음.

**해결**: 카카오 로컬 REST API로 교체 (응답 ~0.3초). "서울시"→"서울특별시" 자동 정규화, 주소 검색 실패 시 키워드 검색 자동 폴백. 카카오 앱 생성 시 카카오맵 서비스 활성화 및 플랫폼 등록 필요함을 확인.

---

### 🔴 난관 4. 반경 내 충전소 탐색 성능
**문제**: 7만 건 충전기 데이터에 대해 `apply()` 파이썬 루프로 Haversine 거리 계산 → 검색마다 수만 번 순회로 느림.

**해결**: numpy 벡터 연산으로 전체 일괄 계산하도록 변경 → 즉시 처리.

---

### 🔴 난관 5. 행정동 지도에 비서울 지역 표시 버그
**문제**: 행정동별 모드 선택 시 인천 등 비서울 지역이 지도에 표시됨.

**해결**: `get_seoul_dong_geojson()` 함수가 캐시 파일을 필터링 없이 그대로 읽는 문제. 서울 25개 자치구 목록(`SEOUL_GU`)으로 feature 필터링 추가.

---

### 🔴 난관 6. 상일동 미세 버킷 오탐
**문제**: 강동구 상일동이 부족지수 20.5로 TOP에 등장. 실제로는 행정구역 개편으로 생긴 잔여 버킷(전기차 41대).

**해결**: 인근 상일제1동(607대/충분)·상일제2동(157대/충분)과 합산 시 지역 전체는 충분. TOP 목록에서 제외하고 발표 자료에 사유 명시.
